# Windows System Health Check

## Project Overview

This project documents a basic health check of my Windows laptop using built-in Windows tools. The aim was to understand how IT Support Technicians gather system information, assess device performance and identify potential issues before attempting to troubleshoot them.

---

## Objectives

- Assess system performance using Task Manager.
- Review installed hardware using Device Manager.
- Collect network information using Command Prompt.
- Improve my understanding of common Windows administration tools.
- Practise documenting technical findings using GitHub and Markdown.

---

## Tools Used

- Windows Task Manager
- Windows Device Manager
- Command Prompt
- GitHub
- Markdown

## Tasks Completed

### 1. Performance Monitoring

Using Task Manager, I monitored my laptop's CPU, memory and disk usage.

### Observations

- CPU usage remained relatively low during normal use.
- Memory usage stayed between approximately 71–75%.
- I observed that high memory usage can occur even when CPU utilisation is low.
- This demonstrated how RAM availability can affect overall system performance.

---

### 2. Hardware Inspection

Using Device Manager, I reviewed the hardware installed on my laptop.

### Hardware Identified

- AMD Radeon™ Vega 8 Graphics
- WDC PC SN730 SSD
- Realtek 8822CE Wireless LAN Adapter
- AMD Ryzen 5 3500U Processor (8 logical processors displayed)

### What I Learnt

Device Manager enables IT technicians to:

- View installed hardware
- Check driver status
- Investigate hardware faults
- Troubleshoot missing or malfunctioning devices

---

### 3. Network Investigation

Using Command Prompt, I practised several common networking commands.

| Command | Purpose |
|---------|---------|
| `hostname` | Display computer name |
| `whoami` | Display current user |
| `ipconfig` | Display network configuration |
| `ping google.com` | Test internet connectivity |
| `nslookup google.com` | Test DNS resolution |

### Results

- Successfully identified my computer name.
- Verified the logged-in user.
- Confirmed my IPv4 configuration.
- Successfully pinged Google with an average response time of approximately **10 ms**.
- Confirmed that my router was acting as the DNS server.
